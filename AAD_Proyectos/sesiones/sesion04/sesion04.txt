Cuando trabajamos con base de datos, tenemos tres posibles objetos:
	Statement (No lo va a utilizar) -> Ejecuta querys sin tipar -> Los tipos son comprobados por el servidor de BD  
		(Es darle el trabajo a la BD... no es recomendable y tambien es peligroso porque pueden realizar inyecciones de querys)
	PreparedStatement -> Ejecuta una query tipada -> Los tipos se los doy al servidor de forma correcta 
		(va parametriza y se ejecuta mas rapido en el servidor)
		Método para trabajr con PreparedStatemet:
		=> execute() -> me retorna si hay resulset o no, es decir, si hay un conjunto de elementos o no.
			=> Suele utilizar: INSERT
		=> executeUpdate() -> me retorna el número de filas afectadas
			=> Suele utilizar: UPDATE / DELETE
	* Nota : Para la obtencion de datos es diferente. Se utiliza el resultset
	ResultSet => Obtengo la tupla de datos de una tabla. Para ello necesito tener un:
		-> PreparedStatement -> utilizamos <executyQuery()> -> con lo que obtener un <ResultSet>
